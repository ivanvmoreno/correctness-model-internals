\headerbox{\bf Overview}{name=overview,column=0,row=0,span=1}{
\begin{itemize}
\item \textbf{Question:} can an LLM anticipate whether it will answer a question correctly \emph{before} it generates any answer tokens?
\item We extract the residual stream at the final question token and learn a simple linear probe from question-only activations.
\item Probes are trained on TriviaQA and evaluated in distribution and \OOD{} on factual and arithmetic datasets.
\end{itemize}

\vspace{0.3em}
\textbf{\color{accent}Headline findings}
\begin{itemize}
\item A single question-only direction beats the best baseline on \best{24/30} \OOD{} model-dataset comparisons.
\item On factual \OOD{} datasets (People, Cities, Medals), it wins on \best{17/18} comparisons.
\item Best layers appear in the \best{middle-to-late} part of the transformer stack.
\item ``I don't know'' responses lie at the negative extreme of the learned direction.
\item Transfer breaks on GSM8K, suggesting factual and arithmetic correctness are structurally different.
\end{itemize}

\begin{center}
\fcolorbox{border}{accentlight}{
\parbox{0.92\linewidth}{
\centering
\textbf{\color{ink}Takeaway:} LLM internals contain a linearly accessible \emph{pre-answer self-assessment signal} for factual QA.
}}
\end{center}
}

\textbf{\color{accent}Datasets}
\begin{itemize}
\item \textbf{Training source:} TriviaQA (60K), chosen for breadth and diversity.
\item \textbf{Factual \OOD{} tests:} Cities (10K), Notable People (16K), Medals (9K).
\item \textbf{Arithmetic tests:} Math Operations (6K) and GSM8K (8K).
\end{itemize}

\textbf{\color{accent}Protocol}
\begin{itemize}
\item 3-shot prompting, temperature 0, activations captured at the last prompt token.
\item 10K TriviaQA samples are reserved for layer selection; the remaining data uses 5-fold evaluation.
\item After activations are cached, training the probe is a one-shot operation taking less than 3 minutes on CPU.
\end{itemize}
}

\headerbox{\bf Why It Matters}{name=why,column=0,below=setup,above=bottom,span=1}{
\begin{itemize}
\item The signal is \textbf{internal}: it generalizes better than black-box assessors built from question embeddings.
\item The probe is \textbf{lightweight}: no answer sampling or uncertainty module is required at test time.
\item A model-specific correctness direction could support abstention, routing, or human-review policies for factual QA.
\item The GSM8K failure mode suggests that reasoning accuracy requires a different internal verification mechanism.
\end{itemize}

\vspace{0.4em}
\textbf{\color{accent}Practical message}

The strongest gains appear when we want a cheap, pre-generation estimate of whether a model is likely to know a factual answer.

\vspace{0.5em}
\textbf{Paper and code:} \url{github.com/ivanvmoreno/correctness-model-internals}
}

\headerbox{\bf Method}{name=method,column=1,row=0,span=1}{
\begin{itemize}
\item For each question \(x\), extract residual activations \(h^{(l)}(x)\) at every layer \emph{after} the question is read and \emph{before} any answer token is generated.
\item Generate one answer \(y = M(x)\), grade it as correct or incorrect, and compute a linear direction
\[
v^{(l)} = \mu_{\mathrm{correct}}^{(l)} - \mu_{\mathrm{incorrect}}^{(l)}.
\]
\item Score a new question using
\[
s^{(l)}(x) = \langle h^{(l)}(x), v^{(l)} \rangle
\]
and select the layer with the highest held-out \AUROC{} on TriviaQA.
\item Evaluate that same direction in distribution and on unseen datasets.
\end{itemize}

\headerbox{\bf Where The Signal Appears}{name=layers,column=1,below=method,above=bottom,span=1}{
\textbf{\color{accent}Layer-wise emergence}
\begin{itemize}
\item Early layers are weak predictors; performance rises and then saturates in the middle or late-middle of the network.
\item This pattern is consistent across all six model families and scales.
\end{itemize}

\textbf{\color{accent}Sample efficiency}
\begin{itemize}
\item About \best{160} training questions already give strong performance.
\item Around \best{2,560} training questions match the performance of the full 48,540-sample TriviaQA training set.
\item Larger models converge faster, suggesting their correctness signal is more linearly accessible.

\headerbox{\bf What The Direction Captures}{name=qualitative,column=2,below=results,above=bottom,span=2}{
\begin{minipage}[t]{0.58\linewidth}
\begin{itemize}
\item Some models output ``I don't know'' even when prompted to answer in a concrete format.
\item Those abstentions sit at the \textbf{most negative} end of the correctness direction, below ordinary wrong answers.
\item High-score mistakes are often near misses, such as being off by a year; high-score correct answers tend to be canonical, well-known facts.
\item The learned direction therefore behaves like both a \textbf{correctness predictor} and a \textbf{confidence / knowledge-awareness axis}.
\end{itemize}

\vspace{0.4em}
\textbf{\color{accent}Bottom line}
\begin{itemize}
\item LLMs encode a transferable pre-answer factual self-assessment signal in their residual stream.
\item The signal is linearly accessible, data-efficient, and stronger in larger models.
\item Arithmetic reasoning is different: a factual correctness direction does \emph{not} transfer to GSM8K.
\end{itemize}
